# sendPhoto
Mengirim Dokumen dengan lokasi file/url/buffer/fileId/[MediaPhoto](/beta/modules/Utils.Medias.html#MediaPhoto). 
## Parameter
```ts
ctx.telegram.sendPhoto(chatId,photo,more)
```

| parameter | tipe | dibutuhkan | deskripsi | 
| :----: | :---: | :-----: | :---------: |
| chatId | `string` or `bigint` | Ya | Chat/Groups/Channel id. |
| photo | `string` or `Buffer` or [MediaPhoto](/beta/modules/Utils.Medias.html#MediaPhoto) | Ya | Photo yang akan di kirimkan |
| more | JSON objek dari [Parameter Tambahan](#parameter-tambahan) | opsional | Parameter tambahan yang akan di gunakan untuk menjalankan fungsi ini. | 

### Parameter Tambahan
Parameter ini memperpanjang [sendMediaMoreParams](/methods/Media/sendMedia#parameter-tambahan).

| params | type | description | 
| :----: | :---: | :---------: |
| workers | number | Berapa banyak pekerja yang digunakan untuk mengunggah file. apa pun di atas 16 tidak stabil. |
| onProgress | [onProgress](/beta/interfaces/Telegram.Media.onProgress.html) | progress callback saat mengunggah photo | 
| download | boolean | Ketika dokumen adalah url, paksa untuk mengunduh file itu terlebih dahulu sebelum mengunggah ke telegram. defaultnya adalah `false` |

## Contoh
```ts highlight="3"
bot.on("message",async (ctx) => {
     if(ctx.media && ctx.media["_"] == "photo"){
         let hasil = await ctx.telegram.sendPhoto(ctx.chat.id,ctx.media.fileId)
         console.log(hasil)
     }
})
```
## Kemungkinan Kesalahan
Sama seperti [sendMedia](/methods/Media/sendMedia#possible-error)
