# sendAnimation
Sending Video (gif) with file location/url/buffer/fileId/[MediaAnimation](/beta/modules/Utils.Medias.html#MediaAnimation). 
## Parameters 
```ts  
ctx.telegram.sendAnimation(chatId, animation,more)
```

| params | type | required | description | 
| :----: | :---: | :-----: | :---------: |
| chatId | `string` or `bigint` | true | Chat/Groups/Channel id. |
| animation | `string` or `Buffer` or [MediaAnimation](/beta/modules/Utils.Medias.html#MediaAnimation) | true | Video Animation will sending |
| more | JSON object of [More Parameters](#more-parameters) | optional | More parametes will use to execute this function. | 

### More Parametes 
This parameter is extending [sendMediaMoreParams](/methods/Media/sendMedia#more-parameters).

| params | type | description | 
| :----: | :---: | :---------: |
| workers | number | How many workers to use to upload the file. anything above 16 is unstable. |
| mimeType | string | Mime type of video |
| onProgress | [onProgress](/beta/interfaces/Telegram.Media.onProgress.html) | progress callback when uploading video animation | 
| download | boolean | When animation is url, force to downloading that file first before uploading to telegram. default is `false` | 
| thumbnail | `Buffer` or `string` | Set the thumbnail of document, it won't work if you pass with fileId. | 
| width | number | width the video animation |
| height | number | height the video animation | 
| duration | number | duration the video animation | 
| supportsStreaming | boolean | make the video animation support streaming. | 

## Example 
```ts hightlight="2"
bot.command("animation",async (ctx) => {
    let results = await ctx.telegram.sendAnimation(ctx.chat.id,"file id here")
    return console.log(results)
})
```
## Possible Error 
Same as [sendMedia](/methods/Media/sendMedia#possible-error)
