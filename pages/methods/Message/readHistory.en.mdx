# readHistory
Marks message history as read.
## Parameters 
```ts 
ctx.telegram.readHistory(chatId,messageId,more)
```
| params | type | required | description | 
| :----: | :---: | :-----: | :---------: |
| chatId | `string` or `bigint` | true | Target user or group. | 
| messageId | number | True | Message id. |
| more | JSON object of [More Parameters](#more-parameters) | optional | More parametes will use to execute this function. | 

### More Parameters 

| params | type | description | 
| :----: | :---: | :---------: | 
| maxId | number | If a positive value is passed, only messages with identifiers less or equal than the given one will be read. | 

## Example 
```ts highlight="2"
bot.command("readHistory",async (ctx)=>{
    let results = await ctx.telegram.readHistory(ctx.chat.id,ctx.id) 
    console.log(results) 
})
```
## Possible Error 
| code | type | description | 
| :---: | :---: | :------:| 
| 400 | CHAT_ID_INVALID | The provided chat id is invalid | 
| 400 | MSG_ID_INVALID | Invalid message ID provided | 
| 400 | PEER_ID_INVALID | The provided peer id is invalid |
