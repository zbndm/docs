# readHistory
Menandai riwayat pesan sebagai telah dibaca.
## Parameter
```ts
ctx.telegram.readHistory(chatId,messageId,more)
```
| parameter | tipe | diperlukan | deskripsi | 
| :----: | :---: | :-----: | :---------: |
| chatId | `string` or `bigint` | Ya | Target pengguna atau grup. | 
| messageId | number | Ya | Message Id | 
| more | JSON objek dari [Parameter Tambahan](#paramete-tambahan) | opsional | Parameter tambahan yang akan di gunakan untuk menjalankan fungsi ini. | 

### Parameter Tambahan
| params | type | description | 
| :----: | :---: | :---------: |
| maxId | number | Jika nilai positif diteruskan, hanya pesan dengan pengidentifikasi kurang atau sama dari yang diberikan yang akan dibaca. | 

## Contoh
```ts highlight="2"
bot.command("readHistory",async (ctx)=>{
     let hasil = await ctx.telegram.readHistory(ctx.chat.id,ctx.id)
     console.log(hasil)
})
```
## Kemungkinan Kesalahan
| kode | ketik | deskripsi |
| :---: | :---: | :------:|
| 400 | CHAT_ID_INVALID | Id obrolan yang diberikan tidak valid |
| 400 | MSG_ID_INVALID | ID pesan yang diberikan tidak valid |
| 400 | PEER_ID_INVALID | ID rekan yang diberikan tidak valid |
